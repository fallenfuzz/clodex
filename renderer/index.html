Stored as JSON in ~/.clodex/library/exec/. A registered command runs a FIXED argv — an agent's [agent:exec <name>] {json} payload is validated against the def's schema and passed only on stdin, never spliced into the command. Grant a command to a seat in the New / Edit Session dialog (Claude only).

New Exec Command

A JSON object with argv (non-empty; argv[0] is the executable), an object schema the payload is checked against, and optional cwd / timeoutMs / maxBytes. A field typed "filename" in the schema is guarded against path traversal. Rejected on save if malformed.

Traffic optimization

Cache-warming, cost telemetry and wire optimizations. Routes each agent session's API traffic through a local proxy process on this machine, started and managed by Clodex (first run sets up a private Python environment; needs a system python3). Applied at spawn — running sessions pick it up on restart; sessions work normally without it.

—

Resume-time transcript bake

When resuming a session, bake its on-disk transcript down to the same set the proxy already strips on the wire, so the replayed prefix is permanently slimmer. Byte-identical to the live wire (it only removes what the wire already drops), so it never busts a warm cache. Fail-safe — resumes the original transcript on any error.

Capture logs

The proxy keeps per-session request captures on disk to power cost and cache-bust forensics. They accrue over time; clearing collapses old sessions to their billing receipts (cost history is preserved — only the detailed request forensics are removed) and clears the probe-traffic bucket. Active, warm, and recent sessions are never touched.

—

Phone access

Serves a chat-style web view of your agent sessions (transcript + send box, no terminal) on this machine only (127.0.0.1). To reach it from your phone, put both devices on a tailnet and run tailscale serve <port>, or use an SSH tunnel — Clodex never opens a network-visible port itself.

—

Claude MCP

Claude Code auto-injects the built-in claude_design connector (~20 tools, ~4k tokens every turn) with no global opt-out. When a session is routed through a strip-capable proxy, the proxy already removes just those tools and keeps your real MCPs — nothing to do here. This fallback is for unrouted sessions only and uses --strict-mcp-config, which is all-or-nothing: it disables every MCP server, including project/user ones. Leave it off unless your unrouted agents use no MCP. Applied at spawn — restart a session to pick up changes.

Default tools for new sessions

Unchecked tools are denied for new Claude sessions by default (you can still adjust per session at create). A consistent set lets sessions share one cached tools segment — leaving rarely-used tools off also trims each prompt. Applies only to new sessions; existing ones are unaffected.

Peered Clodexes

Attach to agent sessions running in a Clodex on another machine: view their terminals live and, after an explicit "Take control", type into them. Give each peer an SSH host (laptop2 or user@laptop2) and Clodex maintains the tunnel itself — needs key-based SSH (no password prompts) and Phone access enabled on the peer. Test & Set Up checks the box over SSH and, if no Clodex is running there yet, offers to install one. The URL field is the manual alternative for tailnets or custom tunnels. Anyone who can reach the peer's endpoint can drive its sessions — treat it like shell access.
